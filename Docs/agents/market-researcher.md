# 🔭 Market Researcher（市場研究）

> **用途**：產業研究助理，針對某投資主題研究產業概況、競品、同業比較，最終產出研究報告。

`產業/主題 → 產業概覽＋競爭格局＋同業 comps＋點子清單 → 研究報告（可選簡報）`

| 家族 | 用戶 | 頻率 | 風險 |
|---|---|---|---|
| 研究與建模 | 分析師／投組經理(PM) | 接到產業或主題研究需求 | 🟡 中 |

## 一、Workflow

```
 產業/主題 + 一句話切角
     │
     ▼
┌────────┐ ┌────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ ┌────────┐
│①界定   │►│②寫概覽 │►│③競爭格局 │►│④同業comps│►│⑤點子   │►│⑥組裝   │
│8-15名  │ │sector- │ │competit- │ │comps-    │ │idea-   │ │報告    │
│        │ │overview│ │ive-anal  │ │analysis  │ │generat │ └───┬────┘
└────────┘ └────────┘ └──────────┘ └─────┬────┘ └────────┘     │
                                          ▼(stop審)             ▼(stop審)
                                                          分析師審 👤
```
> 可比公司分析（comps），用交易倍數比同業　|　刻意不做：單一標的的追蹤更新（用 earnings-reviewer）

**步驟拆解**
1. **定義研究範圍** — 確定研究問題、市場範圍與研究邊界，找出這個領域裡 8–15 家代表公司。
   - 步驟說明：先框定研究問題與邊界，圈定 8–15 家標的公司，作為後續分析的範圍。
2. **市場全景分析** — 整理這個產業的市場規模、產業結構、值得關注的點、主要成長動力與風險。
   - 步驟說明：外部資料源爬蟲，並產出產業總覽報告（`sector-overview` 技能）。
3. **分析競爭格局與同業比較** — 比較產業裡前幾大公司的營收、成長率與市占率，看誰強誰弱。
   - 步驟說明：MCP 資料串接與動態分析——透過 MCP（CapIQ／FactSet）串接資料源，取得競品的營收、成長、市占等數據比較，比較項目可自訂（`competitive-analysis`、`comps-analysis` 技能）。
4. **篩選潛在關注標的** — 根據前面整理好的產業格局與公司比較，挑出最值得關注的幾個名字。
   - 步驟說明：多源資料整合與投資決策推薦——整合前兩步蒐集的資訊，推薦值得關注的標的（`idea-generation` 技能）。
5. **輸出研究報告** — 把研究成果整理成一份報告，需要時再做成簡報。
   - 步驟說明：報告生成——整合所有研究成果，產出市場研究報告，需要時用 `pptx-author` 產簡報。

## 三、技術架構

**技能鏈**
```
sector-overview ──► competitive-analysis ──► comps-analysis ──► idea-generation
產業概覽           競爭格局               同業倍數          點子清單
```
**Skill（共 5 支）**　`sector-overview` 產業概覽 · `competitive-analysis` 競爭格局 · `comps-analysis` 同業倍數 · `idea-generation` 點子清單 · `pptx-author` 做簡報檔
**MCP（2 個）**　`capiq` · `factset`（公開供應商；capiq 可接 repo 內建的 S&P MCP）

> 工具：讀＋**寫檔**＋CapIQ/FactSet MCP　|　模型：opus-4-7

**外掛 vs CMA（兩種安裝）**
```
 外掛版 (真人盯)         CMA版 (雲端·無頭)
 ┌───────────┐          ┌──────────────────────────┐
 │ 單一agent │          │ 主代理 (不寫)            │
 │ 可直接寫檔│          │  ├ sector-reader   讀資料│
 └───────────┘          │  ├ comps-spreader  spread│
                        │  └ note-writer     唯一可寫│
                        └──────────────────────────┘
```
> 外掛版主代理可直接寫檔（研究草稿、風險中），CMA 版仍把寫檔隔離給 note-writer

**跨 agent**　研究與建模家族 ┄ 與〔earnings-reviewer〕〔model-builder〕互補

## 五、導入評估

| 面向 | 評估 |
|---|---|
| **導入風險** | 🟢 低 — 產出僅內部研究草稿，不對外發布、不涉法規簽核或財務過帳；分析師覆核即可攔錯。真正風險在研究品質與引用正確性（已用 `[UNSOURCED]` 規矩控管），非合規風險。 |
| **導入成本** | 🟡 中 — 需訂閱公開資料源（CapIQ／FactSet，買訂閱＋金鑰）、客製 comps 指標定義與品牌簡報範本；不必自建內部系統，整合輕。 |
| **適用單位** | 賣方／買方研究部、投資團隊、財富管理投資研究 |
| **單位中角色** | 研究分析師（下指令＋覆核草稿）· 投組經理（提需求＋消費研究）· 研究助理（草稿加工） |

**最有機會成功的場景**
1. **陌生產業/主題的初步掃描** — 分析師接到不熟的主題，要快速得到一份結構化概覽，省下大量人工蒐集，CP 值最高。
2. **既有覆蓋產業的定期更新** — 季度／年度例行刷新產業格局與同業 comps。
3. **投資委員會前的主題背景包** — 投組經理要在會上談一個主題，臨時需要格局＋同業＋點子清單。
