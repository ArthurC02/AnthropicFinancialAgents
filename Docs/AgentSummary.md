# 🗂️ Agent 總覽 — 十個助理，一條完整的工作鏈

> 這十個 AI 助理不是各自獨立的工具，而是串起一家金融公司「**贏得客戶 → 開戶 → 基金日常營運 → 對出資人交代**」一整段流程的一套班底。下面先用一張表看懂每個助理，再用一個實際故事串起來，最後談導入評估。
>
> 風險標記：🟢 低、🟡 中、🔴 高（越接近「錢」與「法規」，風險越高、把關越嚴）。

---

## 一、十個助理一覽（在做什麼・什麼時候用・風險）

### 🟦 客戶經營與顧問（前台）
| 助理 | 用途 | 什麼時候用 | 風險 |
|---|---|---|---|
| 🤝 **會前準備**（meeting-prep-agent） | 會議前彙整客戶關係、持倉與相關市場動態，產出給顧問的會前資料包 | 見客戶前 | 🟢 |
| 🎯 **投行提案**（pitch-agent） | 蒐集同業比較與估值，產出含估值模型與品牌簡報的提案包 | 要對一個對象做第一版提案 | 🟡 |

### 🟩 研究與建模（前台）
| 助理 | 用途 | 什麼時候用 | 風險 |
|---|---|---|---|
| 🔭 **市場研究**（market-researcher） | 研究產業概況、競爭對手、同業比較，產出研究報告 | 要搞懂一個產業或主題 | 🟡 |
| 📊 **財報追蹤**（earnings-reviewer） | 公司發財報後解讀法說會、更新模型，產出財報後觀點筆記 | 追蹤中的股票發財報了 | 🟡 |
| 🧮 **估值建模**（model-builder） | 從零建立估值模型（現金流折現、槓桿收購、三大報表等），產出可供決策的試算表 | 要一份全新的估值模型 | 🟡 |

### 🟧 基金行政與財務（後台）
| 助理 | 用途 | 什麼時候用 | 風險 |
|---|---|---|---|
| ⚖️ **總帳對帳**（gl-reconciler） | 比對總帳與明細帳找出差異、追查原因，產出例外報告 | 每天／月底要對帳 | 🔴 |
| 📅 **月底結帳**（month-end-closer） | 估列未付費用、結轉餘額、說明差異，產出月結包 | 每月底結帳 | 🔴 |
| 💰 **估值複核**（valuation-reviewer） | 季末複核投資標的估值、計算收益分配，產出給出資人的報告 | 季末估值 | 🔴 |
| 🧾 **對帳單稽核**（statement-auditor） | 對帳單寄給出資人前，逐欄核對基金淨值、揪差異 | 對帳單寄出前的最後把關 | 🔴 |

### 🟥 開戶與審查（中台）
| 助理 | 用途 | 什麼時候用 | 風險 |
|---|---|---|---|
| 🔍 **開戶審查**（kyc-screener） | 解析開戶文件、套用反洗錢規則、比對黑名單，產出給合規人員的審查報告 | 新客戶開戶或定期複審 | 🔴 |

> 共通點：**每個助理都只「起草、建議」，不會自己下決定**——最後一定有人覆核、簽核。

## 二、一個實際故事：晨曦資本的一季

> 晨曦資本是一家綜合金融公司，既做投資銀行顧問、也經營自己的基金。這一季，它要替新基金找一個大型出資人（把錢交給基金去投資的機構），同時照常營運基金。看十個助理各自在哪一刻登場。

```
時間 ───────────────────────────────────────────────────────────►

 ① 前台·贏得業務       ② 中台·開戶     ③ 後台·基金日常     ④ 後台·季末交代
 ────────────────      ──────────      ──────────────      ──────────────
 🤝 會前準備           🔍 開戶審查      ⚖️ 總帳對帳         💰 估值複核
 🔭 市場研究                           📅 月底結帳         🧾 對帳單稽核
 📊 財報追蹤
 🧮 估值建模
 🎯 投行提案
```

### ① 前台：把潛在出資人變成有興趣的客戶
1. **🤝 會前準備**　顧問下週要見一個大學捐贈基金（潛在出資人），助理會前備好「往來歷史＋持倉＋相關市場動態＋建議議程」。
2. **🔭 市場研究**　會中對方問：「AI 基礎建設這主題值得投嗎？」→ 助理隔天交出產業概覽＋競爭格局＋同業比較＋值得關注的名單。
3. **📊 財報追蹤**　名單裡一檔追蹤中的股票剛好發財報 → 助理讀法說會、更新模型、草擬觀點筆記，讓看法保持最新。
4. **🧮 估值建模**　對方想深入評估其中一家公司 → 助理從零建好現金流折現與槓桿收購估值，供投資判斷。
5. **🎯 投行提案**　同時，投行端要對一個併購對象提案 → 助理自動蒐集同業比較、建估值、產出品牌提案簡報。

### ② 中台：資金要進場，先開戶
6. **🔍 開戶審查**　出資人決定出資 → 助理解析開戶文件、跑開戶與反洗錢審查、比對制裁名單與政治人物名單，整理成審查包給合規人員簽核。
   > ⛔ 這一關沒過，後面都不會發生——開戶是資金進場的閘門。

### ③ 後台：基金開始營運
7. **⚖️ 總帳對帳**　資金進場後，每天比對總帳與明細帳，揪出差異、追查原因。
8. **📅 月底結帳**　每到月底，估列未付費用、結轉餘額、說明差異，組裝結帳包。

### ④ 後台：季末對出資人交代
9. **💰 估值複核**　季底複核投資標的估值、計算收益分配——管理團隊的績效分成與出資人各分多少，並備妥出資人報告。
10. **🧾 對帳單稽核**　對帳單寄出前，最後一道把關——逐欄核對基金淨值、揪差異、給「可放行／需再查」。
    > ✅ 過了這關，報告才寄給出資人——一季的循環收尾。

## 三、兩條貫穿全部的設計原則

**原則一：風險越高，越不讓助理自己動手**
```
🟢🟡 前台／研究  → 產出只是內部草稿         → 助理可以自己存檔（較寬鬆）
🔴   後台／合規  → 牽涉法規與帳目正確性     → 助理不能自己存檔，動作嚴格切開、分權
```

**原則二：人永遠是最後拍板的人**
```
所有助理都只「起草、建議」，從不自己決定：
 開戶 → 合規人員決定   過帳 → 控制員核准   發布 → 分析師簽核   寄送 → 主管簽核
```

## 四、導入評估（風險・成本・先後）

> 各助理的完整分析見各別文件「五、導入評估」；這裡只給整體對照與導入順序。

| 助理 | 導入風險 | 導入成本 | 適用單位 | 主要使用者 | 最適合的場景 |
|---|---|---|---|---|---|
| 🤝 會前準備 | 🟢 低 | 🟡 中 | 財富管理前台 | 財富顧問 | 例行客戶會議前的資料準備 |
| 🔭 市場研究 | 🟢 低 | 🟡 中 | 研究部／投資團隊 | 研究分析師 | 不熟的產業/主題要快速上手 |
| 📊 財報追蹤 | 🟡 中 | 🟡 中 | 股票研究部 | 股票研究分析師 | 財報季要同時追很多檔 |
| 🧮 估值建模 | 🟡 中 | 🟡 中 | 投行／研究部 | 建模專員／分析師 | 從零建一份乾淨的估值模型 |
| 🎯 投行提案 | 🟡 中 | 🟡 中偏高 | 投資銀行前台 | 資深投行人員 | 初次接觸客戶的第一版提案 |
| 🔍 開戶審查 | 🔴 高 | 🔴 高 | 法令遵循部 | 合規人員 | 新客戶開戶的初步審查 |
| ⚖️ 總帳對帳 | 🔴 高 | 🔴 高 | 基金行政後台 | 對帳控制員 | 每天例行的帳目核對 |
| 📅 月底結帳 | 🔴 高 | 🔴 高 | 基金行政後台 | 控制員 | 每月例行的月結打底 |
| 🧾 對帳單稽核 | 🔴 高 | 🔴 高 | 基金行政部 | 基金會計 | 對帳單分送前的把關 |
| 💰 估值複核 | 🔴 高 | 🔴 高 | 基金行政／會計 | 基金會計主管 | 季末估值複核與收益分配 |

**一條清楚的規律：前台「低風險、低門檻」，後台「高風險、高門檻」**

```
            導入成本低 ──────────────────► 導入成本高
 風險低  │                   🤝 會前準備    🔭 市場研究
 風險中  │                   📊 財報追蹤  🧮 估值建模  🎯 投行提案
 風險高  │                   🔍 開戶審查  ⚖️ 總帳對帳  📅 月底結帳
         │                   🧾 對帳單稽核  💰 估值複核
```
> 為什麼後台都落在右上角（又貴又難）：它們要接的是公司**內部系統**（總帳、明細帳、基金淨值、投資組合、黑名單、客戶系統），而這套方案只給了空架子、要自己接上（成本高）；又牽涉法規與帳目正確性，錯不得（風險高）。前台靠的是外部市場資料（向資料商訂閱就有），產出又只是內部草稿（有人覆核能攔錯）。

**導入順序建議（先簡單見效，再啃硬骨頭）**

```
第一批　先導入：研究與顧問四支（🤝 會前準備、🔭 市場研究、📊 財報追蹤、🧮 估值建模）
        風險低到中、成本中等，多靠外部訂閱資料、不必接內部系統，最快見效、試錯成本低。

第二批　接著導入：🎯 投行提案
        風險中、但成果會對外，且要大量客製投行的簡報範本與檢查規則，工程量比第一批大。

第三批　最後導入：後台五支（🔍 開戶審查、⚖️ 總帳對帳、📅 月底結帳、🧾 對帳單稽核、💰 估值複核）
        省下的人工最多、價值最高，但門檻也最高：要接內部系統、對齊法遵與財務流程、
        建立可追查的操作紀錄。建議在前兩批累積信任後再上。
```
