# 🎯 投行提案（pitch-agent）— 筆記

它是「客戶經營與顧問」家族,仍然不必接內部系統(MCP 是公開的 capiq),所以技術門檻沒跳;但它是產出最複雜的一支,你會明顯感覺到「客製工程」開始變重——這正是第二批的特徵。

## 為什麼是 pitch-agent（而不是先碰後台）

排序邏輯沒變，還是「先不必接內部系統的」：

```text
pitch-agent     → MCP 只有公開的 capiq，不必自建 → 可以無痛試
後台五支+開戶    → 全要接內部總帳/淨值/黑名單系統 → 壓軸再上
```

它把前台前三支的能力「整合成一個端到端產品」：

```text
研究(概覽) + 建模(DCF/LBO/comps) + 簡報 → 一份完整提案包
```

## 安裝

```text
/plugin install pitch-agent@fsi-local
```

## 第一句指令（可以沿用你熟的散熱主題）

> 使用者：幫我對 Vertiv（VRT）做一份併購提案初稿：拉同業比較與先例交易、建 DCF 與示意 LBO、做一張足球場估值區間圖，最後填進提案簡報

## 這支「新增要注意」的地方（跟前三支差很多）

① 產出物最複雜 = Excel 模型「＋」品牌簡報(前面幾支頂多產一種)
② 有「兩個」人工關卡：建完模型停一次、做完簡報停一次(不是只停一次)
③ 多一張「足球場圖」：把 comps／先例／DCF／LBO 各法的高/中/低疊成橫條圖跟現價對照
④ 簡報每個數字都綁回 Excel 的具名範圍 → 改模型，簡報跟著同步
⑤ 它「沒有對外通訊工具」——刻意不能寄信/聯絡客戶，只能交內部草稿

## 驗收檢查表

```text
☐ 同業比較/先例交易有沒有用「一致的指標定義」、標出離群值？
☐ 模型部分一樣要過：計算格是公式、不打死數字、可追溯回輸入？
☐ 足球場圖有沒有出來，並標上現價對照？
☐ 簡報上的數字，能不能對得回工作簿裡的具名範圍？
☐ 有沒有在「模型後」「簡報後」各停一次等 banker 審？
☐ 它有沒有試圖對外發送？（正確行為：不能,只交內部草稿）
```

## ⚠️ 對你誠實：這支「客製成本」開始變高

技術上 capiq 是公開連接器、不必自建系統——這點仍無痛。
但要發揮它的價值，關鍵在「銀行簡報範本」：

```text
沒教它你公司版型時 → 簡報步驟用通用範本，排版不會是你要的樣子
要它套對 → 用 /ppt-template 指令把你公司的 PPT 版型教給它一次
```

→ 所以這支「跑得起來」很容易，但「跑到能直接用」要先做一次範本客製

跑通這支,前台/顧問全部收完。最後就剩第三批:

## 第三批　後台五支 + 開戶審查（⚖️📅🧾💰🔍）

省的人工最多、價值最高，但門檻也最高：
要接內部系統(總帳/子帳/淨值/投組/黑名單)、對齊法遵與財務流程、
建立可追查的操作紀錄。建議累積前兩批信任後再上。

📌 提醒:從第三批開始,「無痛」結束了——那些 agent 一定要先接內部系統才能做真正的事。等你要碰時跟我說,我會帶你看「最少要補哪些東西才跑得動」,不會讓你一次吞太多。

跑通 pitch-agent 後跟我說,我帶你挑第三批裡最適合先試水溫的那一支(通常建議從風險可控、流程最固定的開始)。
