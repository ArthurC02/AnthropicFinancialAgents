# ⚖️ 總帳對帳（gl-reconciler）— 筆記

## 背景：下一支建議與壓軸

```text
⚖️ 總帳對帳(gl-reconciler) ← 建議下一支
   後台最複雜：要接「2 個」系統(總帳＋子帳)、扇出並行、
   還有獨立 critic 對抗式複核濾誤報
   ↓
🔍 開戶審查(kyc-screener) ← 壓軸
   法遵敏感度最高
```

要我幫你產 gl-reconciler 的樣本檔（一份總帳 GL、一份子帳 subledger，埋幾筆真差異＋幾筆假差異考它的 critic）嗎？

> 使用者：繼續

好，產 gl-reconciler 的樣本檔。這支的考點是它的 critic（對抗式獨立複核）——能不能留住真差異、濾掉假差異。所以我埋了 3 筆真差異（要留）和 3 筆假差異（時間差／FX 進位／重分類，要濾掉），做成 GL、子帳、對帳參考三份檔。

#### E. 提示：差異類型定義（供分類）

- **時間差 Timing**：交易已成交但跨記帳基礎（交易日 vs 交割日），會自然消除。
- **系統落差 System lag / 漏記**：一邊已入帳、另一邊未更新（如已交割賣出但 GL 未沖）。
- **重分類 Reclass**：同一證券、總額一致，只是分類欄位不同。
- **不明 Unknown**：查無佐證，須升級調查。

## 評分重點

```text
🔴 最重要:critic 有沒有「兩邊都做對」？
   ☐ 真差異 3 筆(VRT/XYZ/Cash)全留下、且分類正確(漏記/系統落差/不明)？
   ☐ 假差異 3 筆(GB123/Bund/SPY)全濾掉或標 benign，沒當錯誤升級？

🔴 過度升報(over-report)＝critic 失敗：
   把 GB123 時間差、Bund FX 進位、SPY 重分類當成「真差異」報上去 → 扣分
   (對帳最常見的毛病就是誤報一堆，淹沒真問題)

🔴 漏報(under-report)＝更危險：
   把 VRT/XYZ/Cash 任何一筆放過 → 嚴重扣分

🟡 追根因:有沒有用未交割清單/匯率表/分類對應「找到證據」再判定？
🟡 例外報告:有沒有給可簽核的清單，且標明「永不過帳、控制員簽核」？
🟢 加分:有沒有指出 SPY 雖淨額為 0，但「分類欄位該對齊」仍值得提醒？
```
