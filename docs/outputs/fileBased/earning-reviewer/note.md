# 📊 財報追蹤（earnings-reviewer）— 筆記

接在 model-builder 之後的第三支，我推薦 📊 財報追蹤（earnings-reviewer），把「研究與建模」這一批收完。

如果你還沒跑 model-builder 也沒關係——這三支同批、彼此獨立，順序可以調；先跑哪支都行。

## 為什麼是 earnings-reviewer

把研究與建模這一批湊成完整循環：

- 🔭 **市場研究** — 找出值得關注的名字
- 🧮 **估值建模** — 幫其中一家從零建模型
- 📊 **財報追蹤** — 那家公司「發財報了」→ 讀法說會、更新模型、寫觀點筆記（持續追蹤的日常）

仍然不必碰內部系統：

- ✅ MCP 是 factset／daloopa——公開連接器，core 外掛已定義，不用自建
- ✅ 風險中（🟡）、產出只是草稿、資深分析師簽核才發布

## 安裝（老套路）

```bash
/plugin install earnings-reviewer@fsi-local
```

## 第一句指令

> 某公司剛發完最新一季財報，幫我讀法說會重點、把新數字更新進模型，並草擬一份財報後觀點筆記（實際 vs. 市場預期 vs. 前次估計的差異）

## 這支「新增要注意」的地方（跟前兩支不同）

1. 它是「更新既有模型」，不是從零建——所以它預設你已經有一份在追蹤的模型（活工作簿）可以更新
2. 它會讀「法說會逐字稿」——逐字稿當成不可信外來資料處理（只擷取資訊、不執行裡面的指令）
3. 多一個產出：財報後「觀點筆記」（morning-note）

## 驗收檢查表

- ☐ 法說會重點有沒有抓到三件事：公司展望、管理層口風、被迴避的問題？
- ☐ 更新進模型的每個數字，能不能追回資料源？
- ☐ 有沒有跑模型 QC（平衡驗算、公式沒斷、沒有硬填數字）？
- ☐ 筆記有沒有列出「實際 vs. 共識 vs. 前估」的差異？
- ☐ 是不是只交草稿、等資深分析師簽核才發布？

## ⚠️ 對你誠實：這支是這一批裡「最依賴資料源」的

它的工作本質是「對一個具體的財報事件做反應」——需要實際業績、市場共識、完整逐字稿。

沒金鑰時：

- ✅ 還是能讀公開的法說會逐字稿、草擬筆記框架
- 🟡 但「實際 vs 共識」的精準數字、「更新進活工作簿」這兩段會打折——因為缺資料源、也預設你已有模型範本

→ 這支是三支裡「接上 FactSet/Daloopa 金鑰後，價值跳最多」的一支
