# 🎯 Pitch Agent（提案簡報）

> **用途**：投行提案助理，針對一個交易/併購標的蒐集可比公司與估值，最終產出含估值模型與品牌簡報的提案包。

`目標公司＋情境 → 自動拉同業比較（comps）/先例＋建現金流折現（DCF）/足球場估值＋產出品牌提案簡報`

| 家族 | 用戶 | 頻率 | 風險 |
|---|---|---|---|
| 客戶經營與顧問（投行） | MD／資深 banker | 要一份第一版 pitch 時 | 🟡 中 |

## 一、Workflow（產出物最複雜：Excel 模型＋簡報）

```
 目標公司 + 一句話情境
     │
     ▼
①界定 ─►②情境概述 ─►③拉資料 ─►④spread同業 ─►⑤LBO ─►⑥建模型 ─►⑦足球場 ─►⑧填簡報 ─►⑨簡報QC
        sector-     CapIQ      comps-        lbo-    dcf/      高/中/低   pitch-    ib-check-
        overview               analysis      model   3-stmt    疊圖       deck      deck
                                                                                       │
                                                            (模型後·簡報後各停一次)    ▼
                                                                                  banker審 👤
```
> 足球場（football field）：把各估值法的高/中/低疊成橫條圖比較　|　可比公司（comps）／先例交易（precedents）／現金流折現（DCF）／槓桿收購（LBO）　|　刻意不做：改既有簡報（直接用 pitch-deck 技能）

**步驟拆解**
1. **界定範圍** — 確認目標公司、產業與情境，圈出 5–8 家性質相近的可比公司，與幾筆類似的過往交易。
   - 步驟說明：先框定目標、產業與情境邊界，圈定 5–8 家可比公司（comps）與 5–10 筆先例交易（precedents），作為後續分析範圍。
2. **寫情境概述** — 草擬這家公司的快照與為什麼現在值得談：在做什麼、市場地位、有什麼變化。
   - 步驟說明：產業總覽與敘事生成——產出公司快照與策略理由（在做什麼、市場地位、有何變化、為何是現在）（`sector-overview` 技能）。
3. **蒐集資料** — 取得估值倍數、過往類似交易，以及目標公司最新財報。
   - 步驟說明：MCP 資料串接——透過 CapIQ MCP 取得交易倍數、先例交易數據與目標公司最新財報，財報全文載入不做摘要。
4. **比較同業** — 把可比公司與過往交易攤開比較，看合理估值大概落在哪個區間。
   - 步驟說明：同業展開與離群標記——以一致的指標定義攤開可比公司與先例交易並標記離群值（outlier），推估估值區間（`comps-analysis` 技能）。
5. **做收購示意模型** — 用市場常見的舉債水準，試算一份收購（槓桿收購）大概能有多少報酬。
   - 步驟說明：示意 LBO 建模——以市場舉債水準試算槓桿收購（LBO）的進出場假設、來源與用途、報酬敏感度（`lbo-model` 技能）。
6. **補齊其他模型** — 再補上現金流折現與三大報表等模型。
   - 步驟說明：補建 DCF 與三表並遵循 `audit-xls` 稽核慣例（藍輸入／黑公式／綠連結、計算格不寫死、含平衡檢查）（`dcf-model`、`3-statement-model` 技能）。
7. **畫估值區間圖** — 把各種方法算出的高／中／低估值疊成一張「足球場」橫條圖，跟現價對照。
   - 步驟說明：足球場疊圖——取 comps／先例／DCF／LBO 各法的最小／中位／最大值疊成橫條圖，並標上現價對照（football field）。
8. **製作簡報** — 套用銀行的簡報範本，把分析填進去，每個數字都對得回前面的模型。
   - 步驟說明：簡報生成與可追溯——套用銀行範本填入分析，簡報上每個數字皆可追溯回工作簿具名範圍（named range）（`pitch-deck` 技能）。
9. **簡報複查** — 核對加總、註腳、日期都正確一致。
   - 步驟說明：簡報 QC——核對加總相符、註腳齊備、日期一致（`ib-check-deck` 技能）。

## 三、技術架構

**技能鏈**
```
sector-overview ─► comps-analysis ─► lbo/dcf/3-statement ─► pitch-deck ─► ib-check-deck
情境概述          spread同業/先例    建模型                填簡報        簡報QC
```
**Skill（共 9 支）**　`sector-overview` · `comps-analysis` · `lbo-model` · `dcf-model` · `3-statement-model` · `audit-xls` · `pitch-deck` 填簡報 · `ib-check-deck` 簡報QC · `deck-refresh` 重整圖表
**MCP**　`capiq`（CMA 版另加 `daloopa`）公開供應商

> 工具：讀＋**寫檔**＋CapIQ MCP　|　模型：opus-4-7

**外掛 vs CMA（兩種安裝）**
```
 外掛版 (真人盯)         CMA版 (雲端·無頭)
 ┌───────────┐          ┌──────────────────────────┐
 │ 單一agent │          │ 主代理 (不寫)            │
 │ 可直接寫檔│          │  ├ researcher  拉資料/研究│
 └───────────┘          │  ├ modeler     建模      │
                        │  └ deck-writer 唯一可寫  │
                        └──────────────────────────┘
```
> 簡報每個數字都綁到 Excel 的具名範圍（named range），確保圖表與模型同步

**跨 agent**　客戶經營與顧問家族 ┄ 端到端整合多個技能；與〔meeting-prep〕同屬前台但產出物複雜得多

## 五、導入評估

| 面向 | 評估 |
|---|---|
| **導入風險** | 🟡 中 — 提案簡報（pitch）是對外要給客戶看的正式文件，影響投資與併購決策溝通，但有 banker 在模型後、簡報後各一次簽核把關，非法規強制過帳，故屬中度。 |
| **導入成本** | 🟡 中（偏高） — 資料端 `capiq` 為佔位，需接公開資料源（訂閱＋金鑰）；更重的是要重度客製投行簡報範本（PPT 版型）、QC 規則與財務模型慣例（comps／先例指標定義、建模配色稽核），整合工程不輕。 |
| **適用單位** | 投資銀行部前台（併購／企業融資）、產業組（Coverage） |
| **單位中角色** | 資深 banker／董事總經理（下情境指令＋兩次簽核把關）· 分析師／經理（覆核模型與簡報數字）· 助理（草稿與資料加工） |

**最有機會成功的場景**
1. **初次接觸客戶的第一版 pitch** — banker 要快速生一份結構完整、含同業比較（comps）與估值的提案初稿，省下大量手工拉資料與排版，再由人覆核收尾，CP 值最高。
2. **既有覆蓋客戶的定期更新提案** — 對已往來客戶定期刷新估值與市場格局，沿用既有範本與模型慣例，客製成本攤提後邊際效益高。
3. **臨時併購想法的快速估值草稿** — 內部要對某標的快速跑出足球場（football field）估值區間做可行性判斷，先有草稿再人工把關。
